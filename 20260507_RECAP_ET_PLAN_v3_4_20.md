# PARTIE B — BUGS IDENTIFIÉS EN TEST (à corriger v3.4.21)

## B.1 Erreur screenshot 2000px — bloquant Claude in Chrome

**Symptôme :** `messages.X.content.Y.image.source.base64.data: At least one of the image dimensions exceed max allowed size for many-image requests: 2000 pixels`

**Cause :** Claude in Chrome prend des screenshots pour "voir" la page VC. La page VC est trop large (> 2000px). L'erreur bloque toute la session.

**Contournement immédiat (Fred) :** réduire la fenêtre Chrome à ~1200px ou zoom 75% avant de lancer le script.

**Fix dans le script :** ajouter instruction explicite dans le script généré :
```
CONTRAINTE TECHNIQUE : n'utilise pas de screenshot pour vérifier les champs.
Utilise uniquement document.querySelector() pour lire et vérifier les valeurs.
Réduis la fenêtre Chrome à 1200px de large si des erreurs d'image apparaissent.
```

## B.2 Trop d'étapes générées par Claude in Chrome (84 étapes)

**Symptôme :** Claude in Chrome décompose le script en 84 micro-étapes au lieu de suivre les 6 étapes définies.

**Cause :** le script est trop verbeux — Claude in Chrome interprète chaque ligne comme une action distincte.

**Fix :** rendre le script plus directif et condensé. Grouper les actions dans des blocs JS exécutables plutôt qu'en instructions texte.

## B.3 Description tronquée

**Symptôme :** seule une fraction de la description HTML est injectée dans le champ VC.

**Cause probable :** Claude in Chrome ne passe pas le HTML complet à `fillField()` — il le tronque ou l'interprète.

**Fix :** passer la description via une variable JS intermédiaire dans le script :
```javascript
var desc = `[DESCRIPTION_HTML_COMPLETE]`;
fillField('textarea[name="rtip_product_description-0-value"]', desc);
```

**Observation :** Claude in Chrome a bien lu le catalogue et trouvé COJFI + COGEX. Il a commencé par COJFI (vieux listing). Comportement correct selon les règles définies — mais confirme que les vieux vendor codes sans offre active sont bien présents dans le catalogue.

# PARTIE D — RÈGLES MÉTIER VALIDÉES (à graver dans le skill)

## D.1 Workflow SKU — règle absolue

**Ne jamais utiliser le SKU d'Amazon Pilot directement pour construire l'URL VC.**
Le SKU Amazon Pilot est une indication — pas une vérité. Il peut être faux ou différent dans le catalogue VC à cause de l'héritage (changements de lieu de stockage, variations, etc.).

**Séquence obligatoire :** catalogue VC → lire SKU réel → construire URL.

## D.2 Multi-vendor codes

- Un ASIN peut avoir plusieurs vendor codes dans VC
- Plus il y en a, plus le risque de vieux listings est élevé
- Tous doivent être modifiés avec le même contenu
- Si > 2 vendor codes → avertissement à Fred
- Si URL directe échoue → "intervention manuelle nécessaire" + passer au suivant

## D.3 Contrainte connexion

Fred doit être connecté sur son compte master VC avant de lancer le script. Claude in Chrome agit sur la session ouverte — pas de gestion de login.
